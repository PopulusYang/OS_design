项目背景与目标

将操作系统各知识点串联为完整链路：

format→ 创建文件→ cc 编译→ run 运行→ 持久化
  • 文件系统：虚拟磁盘、目录树、权限、硬链接、日志恢复
  • 内核模拟：32 位 RISC VM、进程表、页式内存、时间片调度
  • 用户接口：Shell、C 编译器/汇编器、多用户登录
  • 远程访问：--serve 模式 Web 桌面

技术栈：C17 · POSIX · pthreads · CMake · 无第三方依赖

开发难点（1/2）

Inode 重复分配（TextEdit 建文件 + 终端 cc 后源文件被覆盖）
原因：API/终端子进程 inomap 不同步
修复：inomap.c（跳过已占用 inode）· allocator.c（sync 写 superblock）
Web/终端数据不一致（保存后 ls 看不到）
原因：各进程独立块缓存，脏数据未落盘
修复：allocator.c · buf.c · main.c / web_api.c reload

开发难点（2/2）

cat 二进制导致 Web 终端断开
原因：UPX 含 \0 和控制字符破坏 WebSocket
修复：main.c · web_api.c 二进制检测
cc 输出覆盖源文件
原因：源/输出路径相同或 delete+create 误伤
修复：main.c paths_same · vfs_write_bytes

总结

UPFS 实现从虚拟磁盘到用户程序的完整链路：

文件系统 — 块组 · 动态 Inode · Extent · 日志 · 块缓存
内核 — VM · 进程 · 调度 · 系统调用 · IPC
用户态 — Shell · 多用户 · C 工具链
网络界面 — Web 桌面四应用

C17 · ~15000 行 · macOS/Linux · CMake 构建
