# UPFS 操作系统模拟项目 —— 验收讲解内容

---

## 一、底层存储与数据结构

### 1.1 虚拟磁盘布局

UPFS 在宿主文件系统上使用单个镜像文件（默认 `testimg/vfs_disk.img`）模拟虚拟磁盘，总容量为 **546 个逻辑块 × 512 字节/块 ≈ 273 KB**。

| 区域 | 逻辑块号 | 块数 | 说明 |
|------|----------|------|------|
| 引导块 | 0# | 1 | 存放魔数 `UPFSBOOT`，末尾记录超级块和数据区起始位置 |
| 超级块 | 1# | 1 | 文件系统全局元数据，恰好占满 512 字节 |
| i 节点区 | 2# ~ 33# | 32 | 共 512 个磁盘 i 节点，每个 32 字节 |
| 数据区 | 34# ~ 545# | 512 | 实际文件数据、索引块、目录数据块 |

**关键常量**（定义于 `include/vfs_core.h:20-80`）：

```
BLOCK_SIZE = 512          DISK_INODE_SIZE = 32       DIR_ENTRY_SIZE = 16
INODES_PER_BLOCK = 16     TOTAL_INODES = 512         DIRS_PER_BLOCK = 32
MAX_FREE_BLOCKS = 50      ADDRS_PER_BLOCK = 256      ROOT_INODE_NO = 1
```

### 1.2 超级块（SuperBlock）与成组链接法

超级块是文件系统的"总控室"，占满磁盘 1# 块（512 字节），结构定义于 `include/vfs_core.h:135-146`。

**核心字段：**

```
s_magic              uint32_t   魔数 0x55504653 ("UPFS")，挂载时校验
s_inode_total        uint32_t   i 节点总数（512）
s_inode_free_count   uint32_t   当前空闲 i 节点数
s_block_total        uint32_t   数据块总数（512）
s_block_free_count   uint32_t   当前空闲数据块数
s_free_block_count   uint16_t   空闲块栈有效条目数
s_free_block_chain   uint16_t   成组链接下一组登记块号
s_free_block_stack[] uint16_t   当前组空闲块号栈（最多 50 个）
s_inode_stack_top    uint16_t   空闲 i 节点栈顶索引（0xFFFF 表示空栈）
s_inode_free_stack[] uint16_t   空闲 i 节点号栈
```

**成组链接法**是经典 UNIX 教学模型中的空闲块管理算法：

1. 格式化时将所有空闲数据块按每组最多 50 个划分为若干组
2. 第一组块号直接填入超级块 `s_free_block_stack[]`
3. 后续每组用一个"登记块"串联：登记块的 `[0]` 存放本组可用块数，`[1..m]` 存放块号，`[m+1]` 指向下一组登记块
4. 分配时从栈顶弹出（LIFO）；栈空时从 `s_free_block_chain` 指向的登记块恢复下一组
5. 核心规则：栈中仅剩 1 个块时，若该块没有后继链则直接分配，否则将其作为登记块读入下一组

空闲 i 节点采用**栈式管理**：`s_inode_free_stack[]` 存放空闲 i 节点号，`s_inode_stack_top` 为栈顶索引。栈空时扫描磁盘 i 节点区寻找 `d_mode==0 && d_nlink==0` 的节点。

### 1.3 磁盘 i 节点（DiskINode）与混合索引

每个文件/目录对应一个磁盘 i 节点，固定 32 字节（`include/vfs_core.h:160-169`）：

```
d_mode       uint16_t   文件类型（IFDIR=0040000 / IFREG=0100000）OR 权限位
d_nlink      uint16_t   硬链接计数
d_size       uint32_t   文件逻辑长度（字节）
d_uid        uint16_t   所有者用户 ID
d_gid        uint16_t   所有者组 ID
d_direct[8]  uint16_t   8 个直接索引块号（8 × 512 = 4 KiB）
d_sindirect  uint16_t   一级间接索引块号（256 × 512 = 128 KiB）
d_dindirect  uint16_t   二级间接索引块号（256 × 256 × 512 ≈ 32 MiB）
```

**混合索引布局**通过三层索引实现从小文件到大文件的高效空间利用：小文件仅用直接索引即可覆盖 4 KiB，中等文件通过一级间接扩展至 128 KiB，大文件通过二级间接可达约 32 MiB。每个索引块可存放 256 个 16 位块号（`ADDRS_PER_BLOCK = 512/2 = 256`）。

### 1.4 目录项（DirEntry）

目录文件由定长 16 字节的目录项组成（`include/vfs_core.h:174-177`），每块可容纳 32 个目录项：

```
de_inode    uint16_t   关联的 i 节点号（0 表示空闲目录项）
de_name     char[14]   文件名（最多 14 字符，定长域，'\0' 填充）
```

### 1.5 内存 i 节点缓存（MemINode + Hash）

为减少磁盘 I/O，UPFS 实现了内存 i 节点缓存层（`src/fs/allocator.c`）：

- **MemINode 结构**（`include/vfs_core.h:202-208`）：在 DiskINode 基础上增加 `m_inode_no`（磁盘 i 节点号）、`m_ref_count`（引用计数）、`m_flags`（MINODE_DIRTY / MINODE_LOCKED）
- **Hash 表**：128 个桶，以 `ino % 128` 为键，链地址法解决冲突
- **对象池**：预分配 512 个 `IHashNode` 槽位，每个内嵌 `pthread_rwlock_t` 读写锁保证并发安全
- **iget(ino)**：Hash 命中则 ref++ 返回；未命中则分配新槽位、读盘、挂链、ref=1
- **iput(ip)**：ref 递减；归零时若 dirty 则回写磁盘，若 `d_nlink==0` 则回收 i 节点号并卸链
- **并发控制**：`inode_rdlock` / `inode_wrlock` / `inode_unlock` 基于 pthread 读写锁

### 1.6 物理内存管理

定义于 `src/kernel/memory.h`，实现于 `src/kernel/memory.c`：

| 参数 | 值 | 说明 |
|------|-----|------|
| 总内存 | 128 MB | 线性字节数组 |
| 页大小 | 4 KB (4096) | 标准页框 |
| 总页数 | 32768 | 128MB / 4KB |
| 内核预留 | 前 4096 页 (16 MB) | 物理地址 0x00000000 ~ 0x01000000 |
| 用户可用 | 后 28796 页 (≈112 MB) | 物理地址 0x01000000 ~ 0x08000000 |

- **位图管理**：每个物理页框用 1 bit 标记空闲/已用
- **API**：`mem_alloc_pages(n)` 分配连续物理页框，`mem_free_pages(page, n)` 释放，`mem_page_ptr(page)` 获取物理页字节指针
- **地址翻译**：`mem_addr_to_page` / `mem_page_to_addr` / `mem_page_offset` 实现物理地址与页号的互转
- **物理内存访问**：`mem_read32` / `mem_write32` / `mem_read8` / `mem_write8` 支持小端读写

### 1.7 进程虚拟地址空间与页表

每个进程拥有独立的虚拟地址空间，最大 4096 页（16 MB），虚拟地址从 `0x00000000` 开始：

```
0x00000000 ───────────────── 文本段（.text，代码）
           ───────────────── 数据段（.data，已初始化数据）
           ───────────────── BSS 段（.bss，未初始化数据，零填充）
           ───────────────── 堆（heap，sbrk 动态增长 ↑）
           ...              ...
0x01000000 ───────────────── 栈顶（stack，向下增长 ↓，默认 16 KB）
```

页表 `p_page_table[MEM_MAX_PROCESS_PAGES]` 将虚拟页号映射到物理页框号，`cpu_virt_to_phys()` 在每次内存访问时进行地址翻译。

## 二、核心文件与目录操作

### 2.1 磁盘块级 I/O（disk_io）

`src/fs/disk_io.c` 是文件系统的最底层，在宿主内存中维护一块与虚拟磁盘等大的字节数组，提供：

- **`disk_create()`**：在内存中 calloc 空白虚拟盘
- **`disk_load(path)`**：从宿主机镜像文件加载到内存
- **`disk_save(path)`** / **`disk_sync()`**：将内存镜像写回宿主机文件（持久化）
- **`read_block(no, buf)` / `write_block(no, buf)`**：以 512 字节块为单位的读写

### 2.2 格式化（format）

`src/fs/format.c` 实现 `mkfs` 功能，初始化全新的空文件系统：

1. 写入引导块（0#），标记 `UPFSBOOT` 魔数
2. 清零整个 i 节点区（32 块）
3. 构造空闲数据块序列（共 511 个，跳过根目录占用的 DATA_ZONE_START），初始化成组链接
4. 初始化空闲 i 节点栈（i 节点号 2~511 压栈）
5. 写入超级块（1#）
6. 创建根目录（i 节点 1，数据块 DATA_ZONE_START），含 `.` 和 `..` 两个目录项
7. 将内存镜像保存到宿主机文件

### 2.3 块分配与回收（balloc / bfree）

基于成组链接法实现（`src/fs/allocator.c:525-605`）：

**balloc() 分配流程：**
1. 栈空则调用 `reload_free_block_stack()` 从登记块链恢复
2. 栈仅剩 1 个时调用 `maybe_reload_last_group_leader()` 判断是否需要作为组长块读入下一组
3. 从栈顶弹出块号，`s_block_free_count--`

**bfree(blk) 回收流程：**
1. 若栈未满（< 50）：直接将块号压入栈顶
2. 若栈已满（= 50）：将当前栈的全部 50 个块号写入被回收块 b，使 b 成为新登记块，清空超级块栈仅保留 b 自身

### 2.4 i 节点分配与回收（ialloc / ifree）

基于栈式管理（`src/fs/allocator.c:609-665`）：

- **ialloc()**：优先从超级块栈弹出；栈空则扫描磁盘 i 节点区寻找空闲节点（`d_mode==0 && d_nlink==0`），清零后返回
- **ifree(ino)**：将磁盘 i 节点清零写回，压入超级块栈

### 2.5 路径解析（namei）

`src/fs/dir_sys.c` 实现，支持绝对路径（`/` 开头）和相对路径（基于当前用户 `u_cdir`）：

1. 从根目录或当前目录的 MemINode 开始
2. 逐级按 `/` 分割路径分量
3. 在每个目录中线性扫描 DirEntry，匹配文件名
4. 通过 `iget()` 获取下一级目录/文件的 MemINode
5. 遇到 `..` 则返回父目录，`.` 则保持当前
6. 最终返回目标 MemINode（ref=1，调用者负责 `iput`）

### 2.6 文件操作（file_sys）

`src/fs/file_sys.c` 提供完整的文件 CRUD 接口：

**创建文件 `vfs_create(path, mode)`**：
1. 路径解析获取父目录 MemINode
2. 检查同名文件是否已存在
3. `ialloc()` 分配新 i 节点，设置 d_mode/d_uid/d_gid/d_nlink=1
4. `dir_link_entry()` 在父目录中追加目录项

**打开文件 `vfs_open(path, mode)`**：
1. `namei()` 解析路径
2. 在用户打开文件表 `u_ofile[]` 中分配空闲槽位
3. 根据模式加读锁（`inode_rdlock`）或写锁（`inode_wrlock`）
4. 返回 fd（0 ~ MAX_OPEN_FILES-1）

**读取文件 `vfs_read(fd, buf, count)`**：
1. 根据 fd 找到 `OpenFileTable` 和关联的 MemINode
2. 通过混合索引（直接/一级间接/二级间接）将 `oft_read_pos` 字节偏移翻译为逻辑块号
3. `read_block()` 读取数据，递增读指针

**写入文件 `vfs_write(fd, buf, count)`**：
1. 类似读取，但将 `oft_write_pos` 翻译为逻辑块号
2. 若需写入新块则调用 `balloc()` 分配
3. 必要时扩展一级/二级间接索引块
4. 更新 `d_size`，标记 `MINODE_DIRTY`

**关闭文件 `vfs_close(fd)`**：释放读写锁，`iput` 减少引用计数

**删除文件 `vfs_delete(path)`**：遍历全部索引块释放所有数据块（`bfree`），从父目录删除目录项，`iput` 使 `d_nlink` 归零触发 i 节点回收

### 2.7 目录操作（dir_sys）

- **`vfs_mkdir(path, mode)`**：分配 i 节点和数据块，写入 `.` 和 `..` 目录项，在父目录中链接
- **`chdir(path)`**：解析路径，更新当前用户 `u_cdir`
- **`dir_list(path)` / `ls`**：读取目录数据块，逐项打印文件名、类型、大小、权限

### 2.8 混合索引寻址算法

文件逻辑偏移量到物理块号的翻译是文件系统的核心算法：

1. 偏移 < 8 × 512（4 KiB）：使用 `d_direct[offset/512]` 直接索引
2. 偏移 < 4 KiB + 256 × 512（132 KiB）：读取 `d_sindirect` 索引块，从中取第 `(offset-4096)/512` 个块号
3. 偏移 < 4 KiB + 128 KiB + 256×256×512（≈32 MiB）：先读 `d_dindirect` 获取一级索引块号，再从一级索引块中获取二级索引块号

### 3.2 CPU / VM 执行引擎

`src/kernel/cpu.h` 和 `src/kernel/cpu.c` 实现了完整的 32 位 RISC 虚拟机：

**硬件规格：**
- 16 个通用寄存器（R0~R14 通用，R15 = SP 栈指针）
- PC（程序计数器，指令索引而非字节偏移）
- FLAGS（ZF 零标志位）
- 定长 32 位指令编码：`[opcode:8][rd:4][rs1:4][rs2:4][imm12:12]`

**18 条指令：**

| 类别 | 指令 | 操作码 | 说明 |
|------|------|--------|------|
| 控制 | HALT | 0x00 | 停止执行，触发进程退出 |
| 数据 | MOVI | 0x01 | R[rd] = sign-extend(imm12) |
| 数据 | MOV | 0x02 | R[rd] = R[rs1] |
| 数据 | LUI | 0x15 | R[rd] = imm12 << 12（加载高 20 位地址） |
| 访存 | LD | 0x03 | R[rd] = mem[R[rs1] + imm12] |
| 访存 | ST | 0x04 | mem[R[rs1] + imm12] = R[rs2] |
| 算术 | ADD | 0x05 | R[rd] = R[rs1] + R[rs2] |
| 算术 | SUB | 0x06 | R[rd] = R[rs1] - R[rs2] |
| 算术 | MUL | 0x07 | R[rd] = R[rs1] * R[rs2] |
| 算术 | DIV | 0x08 | R[rd] = R[rs1] / R[rs2] |
| 逻辑 | AND | 0x09 | R[rd] = R[rs1] & R[rs2] |
| 逻辑 | OR | 0x0A | R[rd] = R[rs1] \| R[rs2] |
| 逻辑 | XOR | 0x0B | R[rd] = R[rs1] ^ R[rs2] |
| 比较 | CMP | 0x0C | FLAGS = R[rs1] - R[rs2]（更新 ZF） |
| 分支 | JMP | 0x0D | PC += sign-extend(imm12) |
| 分支 | JZ | 0x0E | if ZF: PC += sign-extend(imm12) |
| 分支 | JNZ | 0x0F | if !ZF: PC += sign-extend(imm12) |
| 栈 | CALL | 0x10 | PUSH PC; PC += imm12 |
| 栈 | RET | 0x11 | POP PC |
| 栈 | PUSH | 0x12 | SP -= 4; mem[SP] = R[rs1] |
| 栈 | POP | 0x13 | R[rd] = mem[SP]; SP += 4 |
| 系统 | SYSCALL | 0x14 | 触发系统调用 n=imm12（R0 传递调用号） |

**执行循环：** `cpu_step(ctx)` 每次取指、译码、执行一条指令，通过 `cpu_virt_to_phys()` 将虚拟地址翻译为物理地址后进行内存访问。

### 3.3 可执行文件格式（.upx）

`src/kernel/process.h:45-52` 定义了 UPX 文件头：

```
[magic:4]      "UPX\0"        魔数标识
[entry:4]      uint32_t       入口指令索引
[text_size:4]  uint32_t       代码段大小
[data_size:4]  uint32_t       数据段大小
[bss_size:4]   uint32_t       BSS 段大小
[stack_size:4] uint32_t       栈大小
[text:N]       ...            代码段
[data:N]       ...            数据段
```

**加载流程 `upx_load(p, data, size)`**：
1. 验证 UPX 头（魔数、入口合法性、文件完整性）
2. 释放进程旧页表
3. 按 text/data/bss/stack 四个段分别分配物理页框并建立页表映射
4. 将 UPX 中的 text/data 逐字节拷贝到对应物理页
5. bss 段零填充
6. 设置 PC = entry、SP = 栈顶 - 4、ticks_left = 100

### 3.4 进程管理

`src/kernel/process.c` 实现了类 UNIX 的进程模型：

**PCB 结构**（`src/kernel/process.h:55-88`）包含：进程 ID、父进程 ID、进程名、状态、CPUContext（寄存器+PC+FLAGS）、页表、段布局信息、打开文件表、当前目录、环境变量、父子关系数组。

**5 种进程状态：** FREE → READY → RUNNING → BLOCKED / ZOMBIE

**核心 API：**

| 函数 | 说明 |
|------|------|
| `proc_alloc()` | 分配 PCB 槽位，分配 PID，初始化页表第 0 页 |
| `proc_free(p)` | 释放所有物理页框、环境变量，状态置 FREE |
| `proc_fork()` | 深拷贝父进程页表（每个已映射页分配新物理页并 mem_copy），复制 CPU 上下文和打开文件表，子进程 R0=0，父进程 R0=子进程 PID |
| `proc_exec(p, path)` | 通过 VFS 读取 .upx 文件，调用 `upx_load` 替换进程映像 |
| `proc_wait(status)` | 遍历子进程列表寻找 ZOMBIE 子进程，回收之 |
| `proc_exit(code)` | 设置退出码，状态置 ZOMBIE，保留 PCB 供父进程 wait 回收 |
| `proc_sbrk(p, inc)` | 扩展进程堆，分配新物理页并建立映射 |

### 3.5 时间片轮转调度器

`src/kernel/scheduler.c` 实现：

- **就绪队列**：环形缓冲区，存储 PCB 索引，FIFO 出队入队
- **时间片**：每个进程 100 条指令（`CPU_TIMESLICE = 100`）
- **`sched_tick()` 调度循环**：
  1. 获取当前进程，若无效则从就绪队列取下一个
  2. 执行 `cpu_step()` 循环直到时间片耗尽或触发 HALT/SYSCALL
  3. 若触发 SYSCALL，调用 `syscall_dispatch()` 处理
  4. 若触发 HALT，调用 `proc_exit(0)` 退出进程
  5. 当前时间片结束后将进程重新入队（若仍 RUNNING）
  6. 挑选下一个就绪进程继续执行
- 在 Shell 中 `run` 命令通过 `while (sched_tick() == 0) {}` 循环调度直到所有进程完成

### 3.7 多用户管理

`src/user/user_mgmt.c` 实现了完整的多用户系统：

**安全特性：**
- 口令哈希：随机 8 字节盐值 + 256 位迭代哈希（10000 轮），输出 64 字符十六进制串
- `/etc/passwd`：用户数据库持久化文件，格式为 `username:uid:gid:home:hash:salt`
- 权限控制：`passwd` 仅 root 或用户本人可操作

**UserAccount 结构：** ua_name、ua_uid（普通用户从 1000 起）、ua_gid、ua_home、ua_passwd_hash、ua_salt

**API：** `user_add` / `user_delete` / `user_verify` / `user_find` / `user_passwd` / `user_create_home`

### 3.8 环境变量系统

`src/user/env.c` 实现了双层环境变量体系：

| 层级 | 文件 | 作用域 |
|------|------|--------|
| 系统级 | `/etc/environment` | 所有用户共享 |
| 用户级 | `~/.env` | 每个用户独立 |

- 格式：`KEY=VALUE`，每行一对
- 查询优先级：用户 env > 系统 env
- 容量：最多 64 个变量，KEY 最长 64 字符，VALUE 最长 256 字符
- shell 命令 `export` / `unset` 操作当前用户的环境变量
- VM 内进程可通过 `GETENV` / `SETENV` / `UNSETENV` 系统调用访问

### 3.9 汇编器

`src/assembler.c` 实现两遍汇编器，将 UPFS 汇编源码翻译为 .upx 可执行文件：

- **第一遍**：符号收集（标签地址、字符串常量、数据定义），建立符号表
- **第二遍**：指令编码（`CPU_ENCODE` 宏），符号回填，生成 UPX 文件头和各段
- Shell 命令 `asm` 通过 VFS↔宿主机桥接：导出源文件到 `/tmp`，调用 `assemble_file()`，导入输出

### 3.10 TCP 多终端服务器

`src/serve.c` 实现了 `--serve` 模式下的多终端服务：

- **端口 8080**：HTTP 服务器，提供终端 Web 页面 + WebSocket 升级（`/ws/*`）
- **端口 4096**（可自定义）：原始 TCP 终端，可通过 `nc` 或 `telnet` 连接
- 每个连接 fork 子进程运行 `upfs_session()`，父进程 poll 并转发 I/O
- **共享磁盘**：多个终端共享同一磁盘镜像，一个终端 mount/format 后其他终端自动感知（通过 `shared_disk_path` / `shared_set_disk`）

### 3.11 系统启动总流程

```
main()
 ├─ --serve? → serve_main(port)
 │    └─ 监听端口 → accept → fork → upfs_session(sock, sock)
 │
 └─ 交互模式 → upfs_session(0, 1)
      ├─ ui_banner() 横幅
      ├─ startup_disk_probe() 扫描磁盘镜像
      ├─ 提示 mount 或 format
      ├─ format → disk_create → 初始化引导块/超级块/空闲块链/根目录
      │         → disk_save → fs_mount → POSIX目录 → 用户初始化
      │         → 注入预置程序 → 注入汇编源码 → 创建首个用户
      ├─ mount → disk_load → 验证魔数 → 重建i节点栈
      │        → mem_init → proc_init → sched_init → env_init
      │        → proc_create_init → user_db_load → 登录提示
      └─ 命令循环：ui_prompt → fgets → parse → dispatch
           ├─ 文件命令 → file_sys → allocator → disk_io
           ├─ 目录命令 → dir_sys → namei → allocator
           ├─ 用户命令 → user_mgmt
           ├─ 环境命令 → env
           ├─ 进程命令 → process + scheduler + syscall + cpu
           └─ run → proc_alloc → proc_exec → sched_enqueue → sched_tick循环
```

## 附录：关键数据流示意图

### 文件读取数据流
```
Shell "cat /etc/passwd"
  → vfs_open("/etc/passwd", O_RDONLY)
    → namei("/etc/passwd") → iget(ino)
    → 分配 OpenFileTable 槽位 → inode_rdlock
  → vfs_read(fd, buf, 512)
    → 混合索引: d_direct[offset/512]
    → read_block(blk_no, buf)
  → vfs_close(fd) → inode_unlock → iput
```

### 程序运行数据流
```
Shell "run /bin/hello"
  → proc_alloc() → 分配 PCB
  → proc_exec(p, "/bin/hello")
    → vfs_open → vfs_read → 读入 UPX 二进制
    → upx_load → 分配物理页 → 拷贝代码/数据 → 设置 CPU 上下文
  → sched_enqueue(p)
  → sched_tick() 循环
    → cpu_step() × N
      → SYSCALL WRITE → syscall_dispatch → vfs_write / fputs
      → HALT → proc_exit(0)
```

### 用户登录数据流
```
Shell "login alice password123"
  → user_verify("alice", "password123")
    → 从 /etc/passwd 读取 salt + stored_hash
    → user_hash_password("password123", salt, computed_hash)
    → 比较 stored_hash == computed_hash
  → 切换 User 上下文 → env_user_load("alice")
  → dir_bind_user → chdir → 设置家目录
```
