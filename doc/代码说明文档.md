# UPFS 项目代码说明文档

本文档汇总与本项目实现相关的全部源代码、头文件、示例程序与构建配置，并逐一说明每个文件的作用、所在层次及主要接口。

---

## 一、项目总览

**UPFS**（UNIX 文件系统模拟器）是一个操作系统课程设计项目，在宿主 Linux/macOS 上通过单个磁盘镜像文件模拟虚拟块设备，实现：

- **文件系统层**：ext2 风格块组、XFS 风格动态 Inode、Extent 数据映射、元数据日志、块缓存、VFS 抽象
- **内核层**：128MB 物理内存、4KB 页分配、32 位 RISC 虚拟机、进程管理、时间片轮转调度、20+ 系统调用、管道与 System V 风格 IPC
- **用户层**：多用户账户（`/etc/passwd`）、环境变量（`/etc/environment`、`~/.env`）
- **Shell 与工具**：交互式命令行、`.s` 汇编器、C 子集编译器、终端编辑器
- **网络服务**（`--serve`）：HTTP 8080 Web 桌面、WebSocket 终端/API、4096 原始终端

**语言**：C17  
**依赖**：libc、pthreads，无第三方库  
**主可执行文件**：`build/bin/OS_design`（CMake）

### 1.1 架构分层

```
┌─────────────────────────────────────────────────────────┐
│  Shell (main.c)  │  Web 桌面 (web_page.h + serve.c)     │
├─────────────────────────────────────────────────────────┤
│  工具链: assembler / compiler / editor / binaries       │
├─────────────────────────────────────────────────────────┤
│  进程管理 ─ 调度器 ─ 系统调用 ─ VM (cpu) ─ 内存 (memory) │
│  管道 (pipe) ─ IPC (ipc) ─ 共享内核 (kernel_shared)      │
├─────────────────────────────────────────────────────────┤
│  VFS (vfs.c) ─ UPFS: dir_sys / file_sys / allocator …  │
├─────────────────────────────────────────────────────────┤
│  块缓存 (buf) ─ 日志 (journal) ─ 块组 (bg) ─ inomap     │
│  extent ─ format ─ disk_io                              │
└─────────────────────────────────────────────────────────┘
                          ↓
                   磁盘镜像文件 (273KB, 546 块)
```

### 1.2 磁盘布局（546 块 × 512B）

| 区域     | 块号        | 说明                         |
|----------|-------------|------------------------------|
| 引导块   | 0           | 魔数标识                     |
| 超级块   | 1           | 全局元数据、块组摘要         |
| 块组区   | 2～513      | 8 组 × 64 块，成组链接空闲表 |
| 日志区   | 514～545    | 32 块元数据日志              |

## 三、公共头文件（`include/`）

### 3.1 `include/vfs_core.h`

**作用**：整个项目的**核心常量与磁盘/运行时数据结构**定义，被 FS、VFS、Shell 共同引用。

**主要内容**：
- 布局常量：`BLOCK_SIZE`(512)、`BG_COUNT`(8)、`BG_BLOCKS_PER_GROUP`(64)、`JOURNAL_ZONE_*`、`DATA_ZONE_BLOCKS` 等
- 磁盘结构：`SuperBlock`、`DiskINode`、`Extent`、`DirEntry`
- 运行时结构：`MemINode`（含 `pthread_rwlock_t`）、`SysOpenFile`、`User`、`OpenFileEntry`
- 文件类型与权限宏：`IFREG`、`IFDIR`、`S_IRUSR` 等
- 默认镜像路径：`DEFAULT_DISK_PATH`

**所属层次**：公共 / 数据结构

---

### 3.2 `include/vfs.h`

**作用**：**虚拟文件系统（VFS）抽象层**，用操作表将 Shell/内核 syscall 与具体文件系统（UPFS）解耦。

**主要内容**：
- 操作表：`inode_operations`、`file_operations`、`super_operations`、`file_system_type`
- 统一 API：`vfs_mount/umount/format/sync`、`vfs_create/mkdir/open/read/write/close/delete`、`vfs_chdir/listdir/stat/chmod/access/copy/link`
- 注册入口：`vfs_upfs_register()`、`vfs_register_filesystem()`

**所属层次**：VFS 抽象层

---

### 3.3 `include/kernel_shared.h`

**作用**：定义跨进程共享的**内核全局状态** `KernelShared`，供 `--serve` 模式下多个子进程（终端/API）共享同一套进程表、内存、调度队列与 IPC 表。

**主要内容**：
- `KernelShared`：物理内存数组、页位图、进程表 `PCB[]`、就绪队列、管道表、IPC 表
- 全局指针：`g_kernel`
- 生命周期：`kernel_shared_create/attach/destroy`、`kernel_local_init/shutdown`

**所属层次**：内核 / 多进程共享

## 四、文件系统层（`src/fs/`）

文件系统自底向上：块设备 → 缓存 → 块组/inode 映射 → extent → 目录/文件 → VFS 注册。

### 4.1 块设备 I/O

| 文件 | 作用 |
|------|------|
| **`disk_io.h` / `disk_io.c`** | 磁盘镜像的创建、加载、保存与块级读写。通过 `mmap` 或内存缓冲区模拟 512B 块设备；提供 `disk_create/load/save/sync/shutdown`、`disk_read_block/write_block`。 |

### 4.2 块缓存

| 文件 | 作用 |
|------|------|
| **`buf.h` / `buf.c`** | 全局块缓存（LRU + 哈希表）。`bread` 读块并加锁引用，`bwrite/bdwrite` 写回，`brelse` 释放引用，`bflush_all` 刷盘；挂载/卸载时 `buf_init/shutdown`。所有 FS 元数据与数据块访问均经此层。 |

### 4.3 元数据日志

| 文件 | 作用 |
|------|------|
| **`journal.h` / `journal.c`** | 元数据日志：格式化日志区、`journal_begin/log_block/commit/abort` 事务、`journal_replay` 挂载回放。目录项、inode、extent 等元数据变更先写日志再提交，保证 crash 一致性。 |

### 4.4 块组管理

| 文件 | 作用 |
|------|------|
| **`bg.h` / `bg.c`** | ext2 风格 **8 个块组**，每组 1 个锚块 + 63 个数据块；锚块内**成组链接法**维护空闲块链表。`bg_balloc_for(ino_hint)` 优先在 inode 所在组分配；`bg_sync` 将内存空闲表写回磁盘。 |

### 4.5 Inode 动态映射

| 文件 | 作用 |
|------|------|
| **`inomap.h` / `inomap.c`** | XFS 风格**动态 Inode**：每个 Inode Chunk 占 1 数据块存 16 个 `DiskINode`；B+ 树维护 inode 号 → (chunk, slot) 映射与 chunk 空闲位图。`inomap_ialloc_for/ifree/lookup` 分配与查找；`inomap_read/write_disk_inode` 读写磁盘 inode。 |

### 4.6 Extent 数据映射

| 文件 | 作用 |
|------|------|
| **`extent.h` / `extent.c`** | 文件逻辑块到物理块的 **Extent** 映射 `(lblk, pblk, len)`。小文件 inline 存于 inode；大文件用 B+ 树。`extent_bmap` 分配/查找逻辑块对应物理块，相邻 extent 自动合并。 |

### 4.7 格式化

| 文件 | 作用 |
|------|------|
| **`format.h` / `format.c`** | **mkfs**：初始化超级块、块组、inomap、日志区、根目录；创建 `/bin`、`/etc`、`/home` 等标准目录；注入 demo 二进制与用户账户；调用 `bg_sync` 保证锚块与 inomap 一致。 |

### 4.8 分配器与 Inode 缓存

| 文件 | 作用 |
|------|------|
| **`allocator.h` / `allocator.c`** | **挂载/卸载**入口：`fs_mount/umount`；块/inode 分配 `balloc/balloc_for/bfree`、`ialloc/ialloc_for/ifree`；**内存 inode 哈希缓存** `iget/iput`（带读写锁）；超级块同步 `fs_sync_superblock/fs_sync_disk/fs_reload_super`；调试输出 `fs_debug_print_*`。 |

### 4.9 目录与路径

| 文件 | 作用 |
|------|------|
| **`dir_sys.h` / `dir_sys.c`** | 路径解析 **`namei`**、当前工作目录 **`chdir`**、**`mkdir`**、**`dir_list`**（ls 底层）、目录项增删 `dir_link_entry/dir_unlink_entry`、路径拆分 `dir_split_path`；绑定当前用户 `dir_bind_user` 用于权限检查。 |

### 4.10 文件操作

| 文件 | 作用 |
|------|------|
| **`file_sys.h` / `file_sys.c`** | UPFS **文件 CRUD** 与**两级打开文件表**：用户 FD 表（每用户 20）→ 系统 OFT（全局 40）→ `iget` inode。实现 `upfs_create/open/read/write/close/unlink/lseek/access/stat/chmod/copy/link` 及 `sys_open_file_init/count`。 |

### 4.11 VFS 桥接

| 文件 | 作用 |
|------|------|
| **`vfs.c`** | 将 UPFS 的 dir/file/allocator 操作填入 `file_system_type`，调用 `vfs_upfs_register()` 注册；所有 `vfs_*` 函数转发到当前已注册文件系统。卸载时刷缓存、关闭 journal。 |

## 五、内核层（`src/kernel/`）

### 5.1 共享内存

| 文件 | 作用 |
|------|------|
| **`kernel_shared.c`** | 使用 `mmap(MAP_SHARED|MAP_ANONYMOUS)` 创建/附着/销毁 `KernelShared` 区域；子进程 `kernel_shared_attach` 后共享同一物理内存与进程表。 |

### 5.2 物理内存

| 文件 | 作用 |
|------|------|
| **`memory.h` / `memory.c`** | 模拟 **128MB 物理内存**，4KB 页、位图分配器。`mem_alloc_pages/free_pages` 分配释放；`mem_page_ptr` 页号转指针；`mem_read/write8/32`、`mem_copy/zero` 供 VM 访问；`mem_debug_print` 可视化页使用。 |

### 5.3 虚拟机 CPU

| 文件 | 作用 |
|------|------|
| **`cpu.h` / `cpu.c`** | **32 位 RISC VM**：18 条指令（MOVI/MOV/LD/ST/算术/比较/跳转/CALL/RET/PUSH/POP/SYSCALL/LUI/HALT）；`CPUContext` 含 16 寄存器、PC、FLAGS、页表；`cpu_step` 单步执行；`cpu_virt_to_phys` 地址转换。 |

### 5.4 进程管理

| 文件 | 作用 |
|------|------|
| **`process.h` / `process.c`** | **PCB** 进程控制块：状态、页表、FD 表、UPX 加载信息。`proc_fork/exec/wait/exit`；**UPX 可执行格式**解析 `upx_validate/load`；`proc_alloc_fd/free_fd`；`proc_sbrk` 堆扩展；`proc_shutdown` 清理全部进程。 |

### 5.5 调度器

| 文件 | 作用 |
|------|------|
| **`scheduler.h` / `scheduler.c`** | **时间片轮转**：每进程 100 条指令时间片；就绪队列环形缓冲 `sched_enqueue/remove`；`sched_tick` 时间片耗尽切换；`sched_cooperate` 协作式让出 CPU。 |

### 5.6 系统调用

| 文件 | 作用 |
|------|------|
| **`syscall.h` / `syscall.c`** | VM 执行 `SYSCALL` 时 **`syscall_dispatch`** 分发。基础 20 号：EXIT/FORK/EXEC/WAIT/GETPID、文件 OPEN/CLOSE/READ/WRITE/SEEK/GETCWD/CHDIR/SBRK、环境 GETENV/SETENV/UNSETENV、文件 CREATE/DELETE/MKDIR/STAT；扩展：管道、IPC（信号量/消息/共享内存/FIFO/信号）、HOST 编辑与汇编等。 |

### 5.7 管道

| 文件 | 作用 |
|------|------|
| **`pipe.h` / `pipe.c`** | 内核**匿名管道**：环形缓冲区、读写端引用计数；`pipe_alloc/free`、`pipe_read/write`；与 `proc_fork` 配合实现 shell 管道。 |

### 5.8 进程间通信（IPC）

| 文件 | 作用 |
|------|------|
| **`ipc.h` / `ipc.c`** | System V 风格 **IPC**：信号量 `semget/semop/semctl`、消息队列 `msgget/msgsnd/msgrcv`、共享内存 `shmget/shmat/shmdt`、命名 FIFO `mkfifo`、信号 `kill/getsig`；`ipc_deliver_signals` 在调度时投递挂起信号。 |

## 七、Shell 入口（`src/main.c`）

**作用**：程序**主入口**与**交互式 Shell**，也是 `--serve` 模式下终端子进程的会话逻辑（`upfs_session`）。

**主要功能**：
- **启动分支**：无参数 → 本地 Shell；`--serve [port]` → 调用 `serve_main`
- **磁盘**：`format` / `mount` / `umount`，自动探测镜像路径
- **目录与文件**：`mkdir` `cd` `pwd` `ls` `create` `write` `cat` `rm` `cp` `ln` `stat` `chmod`
- **用户**：`useradd` `login` `su` `logout` `whoami` `passwd` `users`
- **进程与环境**：`run`（含管道链）、`kill` `ps` `env` `export` `unset` `mkfifo`
- **工具**：`asm`（汇编）、`cc`（C 编译）、`vim`（编辑器）
- **调试**：`design_debug`（super/inodes/blocks/bg/sof/memory/process/all）
- **路径安全**：`normalize_vfs_path`、`paths_same` 防止编译输出覆盖源文件；`cat` 拒绝 UPX 二进制

**关键导出**：`main()`、`upfs_session(in_fd, out_fd)`

**所属层次**：应用 / Shell

## 八、网络服务与 Web 桌面

### 8.1 `src/serve.h` / `src/serve.c`

**作用**：**TCP 多终端守护进程**（`--serve` 模式）。

**端口**：
- **8080**：HTTP + WebSocket（浏览器桌面）
- **4096**（或自定义）：原始 TCP 终端（`nc`）

**架构**：
- 父进程 `poll` 循环，接受连接
- `GET /` → 返回 `WEB_PAGE` 静态 HTML
- `WebSocket /ws/N` → fork 子进程，经 socketpair 桥接 `upfs_session`（交互 Shell）
- `WebSocket /api` → fork 子进程，桥接 `upfs_api_session`（JSON API）
- 共享内存 `g_shared` 传递磁盘镜像绝对路径，供 API 子进程挂载

**关键导出**：`serve_main(port)`、`shared_set_disk` / `shared_disk_path`

### 8.2 `src/web_api.c`

**作用**：Web UI 的 **JSON API 会话**（在独立子进程中运行）。

**支持命令**（WebSocket 一行 JSON）：
- 文件：`ls` `stat` `cat` `mkdir` `create` `rm` `write`
- 监控：`debug`（type: super/process/memory/all）

**特点**：每次请求前 `fs_reload_super` + 缓存失效，与终端子进程写操作保持一致；`cat`/`write` 拒绝二进制 UPX 文件。

**关键导出**：`upfs_api_session(in_fd, out_fd)`

### 8.3 `src/web_page.h`

**作用**：内嵌**单页 Web 桌面 UI**（HTML + CSS + JavaScript 字符串常量 `WEB_PAGE[]`）。

**功能**：
- 启动过渡动画（boot screen）
- 菜单栏：应用菜单、连接状态、**关机**、主题切换、时钟
- 可拖拽窗口：Terminal（多 Tab WebSocket）、Finder（文件树）、Activity Monitor（仪表盘）、TextEdit（读写文件）
- Dock、右键菜单、明暗主题（localStorage）

**所属层次**：前端 / 内嵌资源（编译进 `serve.c`）

## 十五、按验收优先级阅读顺序

若时间有限，建议按以下顺序理解核心实现：

1. **`include/vfs_core.h`** — 数据结构与磁盘布局  
2. **`src/fs/disk_io.c` → `buf.c` → `allocator.c`** — 块设备与挂载  
3. **`src/fs/bg.c` + `inomap.c` + `extent.c`** — 三块分配/映射机制  
4. **`src/fs/dir_sys.c` + `file_sys.c` + `vfs.c`** — 路径与文件 API  
5. **`src/kernel/cpu.c` + `process.c` + `syscall.c` + `scheduler.c`** — VM 与进程  
6. **`src/main.c`** — Shell 命令如何串联以上模块  
7. **`src/serve.c` + `web_api.c` + `web_page.h`** — Web 桌面（若演示 `--serve`）

工具链（`compiler/`、`assembler.c`、`editor.c`）与 `involve_src/` 示例可在掌握核心后再读。
