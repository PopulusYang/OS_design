# UPFS 交付文档

## 一、项目功能总览

UPFS 共实现 **6 大子系统、35+ 条 shell 命令、20 个系统调用、9 个 JSON API**，总代码量约 15000 行 C + 1000 行内嵌前端。

### 1.1 文件系统层（12 项核心功能）


| #   | 功能点            | 实现方式                                          |
| --- | -------------- | --------------------------------------------- |
| 1   | 虚拟磁盘           | 546 块 × 512B 镜像文件，mmap 持久化                    |
| 2   | ext2 风格块组      | 8 组 × 64 块，成组链接法空闲管理                          |
| 3   | XFS 风格动态 Inode | Inode Chunk + Loc/Chunk B+ 树，按需分配，支持叶节点分裂     |
| 4   | Extent 区段映射    | 内联 Extent + B+ 树，相邻区段自动合并                     |
| 5   | 块缓存            | LRU + 哈希双重组织，64 槽，延迟写                         |
| 6   | 元数据日志          | 事务提交 + 挂载回放，32 块日志区                           |
| 7   | VFS 抽象层        | 操作表分发，支持可插拔文件系统                               |
| 8   | 两层打开文件表        | 用户表(20) → 系统表(40) → Inode 缓存                  |
| 9   | 路径解析 (namei)   | 支持绝对/相对路径、`.`/`..`/`~`                        |
| 10  | 目录操作           | 定长目录项(16B)，自动维护 `.`/`..`                      |
| 11  | 文件操作           | create/open/read/write/close/delete/copy/link |
| 12  | UNIX 权限模型      | owner/group/other × rwx，root 免检               |


### 1.2 内核层（7 项核心功能）


| #   | 功能点             | 实现方式                            |
| --- | --------------- | ------------------------------- |
| 13  | 物理内存管理          | 128MB 字节数组，位图式页面分配器(4KB 页)      |
| 14  | 32-bit RISC 虚拟机 | 21 条指令，取指-译码-执行循环               |
| 15  | 进程管理            | PCB、进程表(64)、fork/exec/wait/exit |
| 16  | 时间片轮转调度         | 100 条指令/时间片，就绪队列                |
| 17  | 系统调用接口          | 20 个系统调用(文件/进程/环境变量)            |
| 18  | 管道通信            | 匿名管道 + 命名管道(mkfifo)，最多 8 级      |
| 19  | 进程间通信           | 共享内存、信号机制                       |


### 1.3 用户管理层（4 项核心功能）


| #   | 功能点   | 实现方式                                      |
| --- | ----- | ----------------------------------------- |
| 20  | 多用户账户 | uid/gid 分配，/etc/passwd 存储                 |
| 21  | 口令安全  | 随机盐值 + 万次迭代哈希，不可逆                         |
| 22  | 用户切换  | login/logout/su，会话隔离                      |
| 23  | 环境变量  | 系统级(/etc/environment) + 用户级(~/.env)，文件持久化 |


### 1.4 开发工具链（3 项）


| #   | 功能点     | 实现方式                    |
| --- | ------- | ----------------------- |
| 24  | 两遍汇编器   | .s 源码 → .upx 二进制        |
| 25  | C 编译器   | 词法→语法→AST→寄存器分配→代码生成→汇编 |
| 26  | 内置文本编辑器 | vim 风格，支持 :wq/:q 等      |


### 1.5 网络服务层（4 项）


| #   | 功能点                | 实现方式                 |
| --- | ------------------ | -------------------- |

| 29  | WebSocket JSON API | /api 端点，9 个命令        |

### 1.6 Web 管理界面（3 项）

| 调试  | `design_debug` (`super`/`inodes`/`sof`/`memory`/`process`/`all`) |
| 其他  | `help` `clear` `exit`                                            |

## 二、编译与运行指南

### 2.1 环境要求

- 操作系统：macOS 或 Linux
- 编译器：GCC / Clang（支持 C17）
- CMake 3.10+
- 无第三方依赖

### 2.2 编译

```bash
cd /Users/ac/lab/OS
cmake -B build -S . -DCMAKE_C_FLAGS="-D_GNU_SOURCE"
cmake --build build
```

编译产物：`build/bin/OS_design`

### 2.3 运行模式

**模式一：交互式终端**

```bash
./build/bin/OS_design
```

**模式二：Web 管理界面**

```bash
./build/bin/OS_design --serve
# 浏览器打开 http://localhost:8080
# 或 nc localhost 4096 连接原始终端
```

### 2.4 首次使用流程

```
1. 启动程序
2. 输入 format        ← 格式化虚拟磁盘
3. 输入用户名         ← 创建首个用户
4. 输入密码（两次）   ← 设置用户口令和 root 口令
5. 开始使用           ← 文件系统已挂载，用户已登录
```

### 2.5 再次使用流程

```
1. 启动程序           ← 自动检测到已有磁盘镜像
2. 输入 mount         ← 挂载文件系统（自动回放日志）
3. 输入用户名密码     ← 登录验证
4. 继续使用           ← 所有数据完好恢复
```
