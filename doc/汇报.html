磁盘总体布局

虚拟磁盘由 546 个物理块构成，每块 512 字节，总容量 273 KB。

区域起始块号块数大小用途
引导块01512 B启动标识和关键块号
超级块11512 B文件系统全局元数据
块组区2 ~ 513512256 KB8 个块组，存放全部文件和目录数据

块组与成组链接法

成组链接法空闲块管理：

  • 锚块维护容量 50 的空闲块号栈
  • 分配 → 栈顶弹出（O(1)）
  • 栈空 → 从链表加载下一组
  • 回收 → 压入栈顶
  • 栈满 → 写入寄存块，形成链表
  • 创新：按 inode 所在块组就近分配
6

inode · Extent · 缓存 · 日志

动态 inode 管理（类 XFS）

  • 不预留 inode 区，按需从数据块分配 inode 块
  • 每块 16 个 inode（32 B/个），最多 8064 个 inode
  • 位置树：inode号 → (块号, 槽位) — 对数时间定位
  • 空闲树：块号 → 16 位空闲位图 — 快速找空槽
  • 另有 128 项回收栈作为快速路径

文件数据映射（Extent）

  • 每条 Extent 仅 8 字节：逻辑块号 + 物理块号 + 连续长度
  • 三级策略：内联(inode 内) → 单叶(62 条) → B+ 树
  • 自动合并相邻区段 — 顺序写入仅需极少记录

块缓存（LRU + Hash）

1024 个缓存槽 = 512 KB

读取 → 先查哈希表，命中直接返回

写入 → 延迟写（标记脏位，统一刷盘）

元数据日志（WAL）

日志区 32 块：1 块日志头 + 31 块数据槽

修改元数据 → 先写日志 → 标记提交 → 回写原位

崩溃恢复 → 挂载时检测未完成事务 → 自动重放

保证文件系统的崩溃一致性

目录 · 打开文件表 · VFS

目录项（16 B）

inode 号(2B) + 文件名(14B)

每块 32 个目录项

支持 . / .. / ~ / 绝对/相对路径

两级打开文件表

用户表：每用户 20 项（独立读写位置）

系统表：全局 40 项（引用计数）

inode 缓存：2048 项，128 桶哈希 + 读写锁

六种 IPC 机制

匿名管道 ×16

512 B 环形缓冲区，引用计数管理读/写端，空管道自动阻塞。Shell 支持 2~8 级串联。

命名管道 FIFO ×8

通过路径标识，复用匿名管道机制，不相关进程可通过路径通信。

信号量 ×16

键值获取，P(等待/减一) / V(释放/加一)，P 为零时自动让出 CPU。

消息队列 ×8

每队列最多 32 条消息(128B/条)，支持按类型接收。

共享内存 ×8

每段最多 4 页(16 KB)，映射到虚拟地址空间，多进程直接访问。

信号机制

SIGKILL / SIGTERM / SIGUSR1，位掩码挂起，调度时投递。

磁盘格式化 format()

int format(const char *disk_path)
{
    SuperBlock sb;

    if (disk_path == NULL || disk_path[0] == '\0') {
        FORMAT_LOG("错误：磁盘路径为空");
        return -1;
    }

    if (disk_create() != 0) {
        FORMAT_LOG("错误：内存分配失败");
        return -1;
    }

    if (init_boot_block() != 0) {
        FORMAT_LOG("错误：引导块写入失败");
        disk_shutdown();
        return -1;
    }

    if (bg_format_init() != 0) {
        FORMAT_LOG("错误：块组初始化失败");
        disk_shutdown();
        return -1;
    }

    if (journal_init_format() != 0) {
        FORMAT_LOG("错误：日志区初始化失败");
        disk_shutdown();
        return -1;
    }

    if (inomap_format_init(ROOT_DIR_BLOCK) != 0) {
        FORMAT_LOG("错误：动态 inode 初始化失败");
        disk_shutdown();
        return -1;
    }

    memset(&sb, 0, sizeof(sb));
    sb.s_magic = VFS_MAGIC;
    if (bg_fill_super(&sb) != 0) {
        FORMAT_LOG("错误：超级块汇总失败");
        disk_shutdown();
        return -1;
    }

    if (bg_sync() != 0) {
        FORMAT_LOG("错误：anchor 同步失败");
        disk_shutdown();
        return -1;
    }

    if (disk_write_block(SUPERBLOCK_BLOCKNO, &sb) != 0) {
        FORMAT_LOG("错误：超级块写入失败");
        disk_shutdown();
        return -1;
    }

    if (init_root_directory() != 0) {
        FORMAT_LOG("错误：根目录初始化失败");
        disk_shutdown();
        return -1;
    }

    if (disk_save(disk_path) != 0) {
        FORMAT_LOG("错误：镜像保存失败");
        disk_shutdown();
        return -1;
    }

    disk_shutdown();
    return 0;
}

格式化全流程（顺序不可变）

1
分配内存盘 — 在内存中创建 546 块的虚拟磁盘。
2
写入引导块 — 写入启动标识和关键块号。
3
初始化块组 — 建立 8 个块组的锚块和空闲链表。
4
日志区清零 — 初始化 32 块日志区。
5
inode 映射初始化 — 创建位置树和空闲树的根节点，分配根目录的第一个 inode 块。
6
汇总超级块 — 统计空闲块和 inode 数据写入超级块。
7
同步锚块 — 将最新的空闲块状态写回锚块，避免 B+ 树块被误分配。
8
写超级块 — 持久化到块 1。
9
创建根目录 — 在固定位置创建 / 目录，包含 . 和 .. 条目。
10
保存镜像 — 将内存盘写入磁盘文件。

注意：步骤顺序不可变——块组先于 inode 树（后者需要分配块），锚块同步在 inode 树之后（否则空闲栈会包含已占用的块号）。

块分配 bg_balloc_for()

int bg_balloc_for(uint16_t ino_hint)
{
    int start = bg_hint_to_group(ino_hint);
    int bg;
    BgRuntime *rt;
    uint16_t blk;

    for (int pass = 0; pass < BG_COUNT; pass++) {
        bg = (start + pass) % BG_COUNT;
        rt = &g_bg[bg];
        if (rt->block_free == 0)
            continue;
        if (rt->free_stack_count == 0) {
            if (bg_reload_free_stack(rt) != 0)
                continue;
        }
        rt->free_stack_count--;
        blk = rt->free_stack[rt->free_stack_count];
        rt->block_free--;
        return (int)blk;
    }
    return -1;
}

按 inode 就近分配空闲数据块

1
计算起始块组 — 通过 inode 号查找其所在物理块，推算块组编号。尽量在同一块组分配数据块以提升局部性。
2
循环 8 个块组 — 从起始块组出发，取模轮转遍历所有块组。
3
跳过空组 — 若当前块组空闲块为 0，直接尝试下一组。
4
栈为空时重装 — 从链表中读取寄存块，将一组块号填充到栈中。
5
弹出栈顶 — 取出栈顶块号作为分配结果，更新计数。

性能：常态 O(1) 栈弹出，栈耗尽时一次磁盘读取（平均每 50 次分配一次额外 I/O）。

路径解析 namei()

MemINode *namei(const char *path)
{
    MemINode   *ip;
    const char *p;
    char        comp[MAX_FILENAME_LEN + 1];
    uint16_t    start_ino;

    if (path == NULL || g_cur_user == NULL
        || fs_get_superblock() == NULL)
        return NULL;

    start_ino = (path[0] == '/')
        ? ROOT_INODE_NO : g_cur_user->u_cdir;
    ip = iget(start_ino);
    if (ip == NULL) return NULL;

    if (path[0] == '/') {
        p = path_skip_slash(path + 1);
        if (*p == '\0') return ip;
        p = path + 1;
    } else {
        p = path;
    }

    while (*p != '\0') {
        uint16_t next_ino;
        p = path_next_component(p, comp);
        if (p == NULL) { iput(ip); return NULL; }
        if (comp[0] == '\0') break;
        if (!inode_is_dir(ip))
            { iput(ip); return NULL; }

        if (strcmp(comp, ".") == 0) {
            if (path_is_last_component(p))
                return ip;
            p = path_skip_slash(p);
            continue;
        }

        if (strcmp(comp, "..") == 0) {
            MemINode *parent_ip;
            if (dir_lookup(ip, "..", &next_ino) != 0)
                { iput(ip); return NULL; }
            parent_ip = iget(next_ino);
            iput(ip);
            ip = parent_ip;
            if (ip == NULL) return NULL;
            if (path_is_last_component(p))
                return ip;
            p = path_skip_slash(p);
            continue;
        }

        if (dir_lookup(ip, comp, &next_ino) != 0)
            { iput(ip); return NULL; }

        if (path_is_last_component(p)) {
            MemINode *target = iget(next_ino);
            iput(ip);
            return target;
        }

        {
            MemINode *next_ip = iget(next_ino);
            iput(ip);
            ip = next_ip;
            if (ip == NULL) return NULL;
        }
        p = path_skip_slash(p);
    }
    return ip;
}

路径字符串 → 内存 inode 指针

1
确定起点 — 绝对路径从根 inode 开始，相对路径从当前工作目录开始。
2
逐分量循环 — 提取路径中下一个分量名（以 / 分隔）。
3
处理 "." — 当前目录，直接跳过。
4
处理 ".." — 查找父目录的 inode 号，切换到父目录。
5
普通名称 — 在当前目录的 inode 中查找该名称的目录项，获取子项的 inode 号。
6
引用计数管理 — 进入新目录时 iget()获取引用，离开旧目录时 iput()释放引用，保证 inode 缓存的正确性。

所有文件操作的第一步：将路径转换为 inode。

虚拟机单步执行 cpu_step()

int cpu_step(CPUContext *ctx)
{
    if (ctx == NULL) return 1;
    ctx->ticks_left--;

    uint32_t inst = cpu_read32(ctx, ctx->pc * 4);
    uint32_t op  = CPU_OPCODE(inst);
    uint32_t rd  = CPU_RD(inst);
    uint32_t rs1 = CPU_RS1(inst);
    uint32_t rs2 = CPU_RS2(inst);
    int32_t  imm = CPU_IMM12(inst);
    uint32_t *r = ctx->regs;

    switch (op) {
    case OP_HALT:
        ctx->sycall_halt = 1;
        ctx->syscall_no = SYSCALL_EXIT;
        return 1;
    case OP_MOVI:
        if (rd < CPU_NUM_REGS)
            r[rd] = (uint32_t)(int32_t)imm;
        ctx->pc++; break;
    case OP_ADD:
        if (rd < 16 && rs1 < 16 && rs2 < 16)
            r[rd] = r[rs1] + r[rs2];
        ctx->pc++; break;
    case OP_LD:
        if (rd < 16 && rs1 < 16) {
            uint32_t addr = r[rs1] + (uint32_t)imm;
            r[rd] = cpu_read32(ctx, addr);
        }
        ctx->pc++; break;
    case OP_CMP:
        if (rs1 < 16 && rs2 < 16)
            ctx->flags = (r[rs1] == r[rs2])
                ? CPU_FLAG_ZF : 0;
        ctx->pc++; break;
    case OP_JZ:
        if (ctx->flags & CPU_FLAG_ZF)
            ctx->pc = (uint32_t)((int32_t)ctx->pc + imm);
        else ctx->pc++;
        break;
    case OP_CALL:
        r[CPU_REG_SP] -= 4;
        cpu_write32(ctx, r[CPU_REG_SP], ctx->pc + 1);
        ctx->pc = (uint32_t)((int32_t)ctx->pc + imm);
        break;
    case OP_SYSCALL:
        ctx->sycall_halt = 2;
        ctx->syscall_no = (uint32_t)imm;
        ctx->pc++; return 1;
    /* ... 其余 14 条指令结构相同 ... */
    default: ctx->pc++; break;
    }

    if (ctx->ticks_left == 0) return 1;
    return 0;
}

单条指令的取指-译码-执行

1
时间片递减 — 每执行一条指令减一，归零时触发调度切换。
2
取指 — 通过 PC 从虚拟内存取出 32 位指令（经页表翻译）。
3
译码 — 提取操作码(8 位)、目标寄存器(4 位)、源寄存器(各 4 位)、立即数(12 位)。
4
执行 — switch 分支：
· 算术指令直接修改寄存器
· 访存指令经页表翻译后读写物理内存
· 跳转指令修改 PC
· SYSCALL 设置中断标志返回
5
返回值 — 0 表示继续执行，1 表示需要上层处理（时间片耗尽 / 系统调用 / 停机）。

进程派生 proc_fork()

int proc_fork(void)
{
    PCB *parent = proc_current();
    if (parent == NULL) return -1;

    PCB *child = proc_alloc();
    if (child == NULL) return -1;

    for (int i = 0; i < MEM_MAX_PROCESS_PAGES; i++) {
        if (parent->p_page_table[i] != 0) {
            int new_page = mem_alloc_pages(1);
            if (new_page < 0)
                { proc_free(child); return -1; }
            child->p_page_table[i] = (uint32_t)new_page;
            uint32_t src = parent->p_page_table[i]
                         * MEM_PAGE_SIZE;
            uint32_t dst = (uint32_t)new_page
                         * MEM_PAGE_SIZE;
            mem_copy(dst, src, MEM_PAGE_SIZE);
        }
    }

    memcpy(&child->p_cpu, &parent->p_cpu,
           sizeof(CPUContext));
    child->p_cpu.ticks_left = CPU_TIMESLICE;

    child->p_ppid = parent->p_pid;
    child->p_text_start = parent->p_text_start;
    child->p_text_pages = parent->p_text_pages;
    child->p_data_start = parent->p_data_start;
    child->p_data_pages = parent->p_data_pages;
    child->p_bss_end    = parent->p_bss_end;
    child->p_heap_brk   = parent->p_heap_brk;
    child->p_stack_top  = parent->p_stack_top;
    child->p_cwd_ino    = parent->p_cwd_ino;
    strncpy(child->p_name, parent->p_name,
            PROC_NAME_LEN - 1);

    for (int i = 0; i < PROC_MAX_FD; i++)
        proc_dup_fd(&child->p_ofile[i],
                    &parent->p_ofile[i]);

    child->p_cpu.regs[0] = 0;

    if (parent->p_child_count < PROC_MAX_COUNT)
        parent->p_children[parent->p_child_count++]
            = child->p_pid;

    parent->p_cpu.regs[0] = child->p_pid;

    sched_enqueue(child);
    return (int)child->p_pid;
}

完整复制父进程，创建独立子进程

1
分配 PCB — 从进程表中获取一个空闲 PCB。
2
复制内存 — 遍历父进程的 4096 项页表，为每个有效虚拟页分配新物理页并逐字节复制内容。子进程获得完全独立的地址空间。
3
复制 CPU 上下文 — 子进程将从与父进程相同的 PC 位置继续执行。重置时间片。
4
复制元数据 — 段地址、堆指针、栈顶、工作目录等全部继承。
5
复制文件描述符 — 管道类描述符增加引用计数，保证管道在任一端关闭后仍可工作。
6
区分返回值 — 子进程 R0=0，父进程 R0=子PID。这是 fork 的经典语义。
7
加入就绪队列 — 子进程入队等待调度。

int extent_bmap(MemINode *ip, uint32_t lblk,
                int create, uint16_t *phys_out)
{
    // 1. 在现有映射中查找
    if (extent_lookup(&ip->m_dinode, lblk, phys_out) == 0)
        return 0;
    if (!create) return -1;
    // 2. 分配新物理块
    blk = balloc_for(ip->m_inode_no);
    // 3. 尝试快速追加（合并到末尾）
    if (extent_try_extend_run(ip, lblk, blk) == 0)
        { *phys_out = blk; return 0; }
    // 4. 构造新 Extent 并插入
    new_e = {lblk, blk, 1};
    insert_extent(ip, &new_e);
    *phys_out = blk; return 0;
}

journal_replay() — 崩溃恢复

int journal_replay(void)
{
    JournalHeader hdr;
    JournalLogRec rec;

    if (journal_read_header(&hdr) != 0)
        return 0;
    // 已提交且有日志 → 需要重放
    if (hdr.jh_committed == 0 || hdr.jh_nlogs == 0)
        return 0;

    for (i = 0; i < hdr.jh_nlogs; i++) {
        disk_read_block(journal_data_block(i), &rec);
        if (rec.lr_type != JTXN_LOG) continue;
        // 将日志中的完整块副本写回原位
        journal_apply_log(rec.lr_block, rec.lr_data);
    }

    hdr.jh_committed = 1;
    hdr.jh_nlogs = 0;
    return disk_write_block(header_block, &hdr);
}

幂等操作：日志保存完整块副本而非增量，多次重放不产生副作用。

成组链接法空闲块管理

标准成组链接法 + 按 inode 所在块组就近分配，提升数据局部性。

块分配算法（按 inode 就近）
1输入：父 inode 编号
2通过位置树查找父 inode 所在物理块 → 计算块组编号
3循环遍历 8 个块组（从该组开始，取模轮转）
4  若空闲块=0，跳过
5  若栈空，从链表寄存块加载一组块号
6  栈顶弹出，空闲块数减一，返回
7所有块组均满 → 分配失败
块回收算法
1根据块号确定所属块组
2若栈未满 → 压入栈顶
3若栈已满(50项) → 将栈内容写入回收块 → 该块成为新寄存块 → 清空栈

动态 inode 分块分配与 B+ 树索引

不预留 inode 区，按需从数据块分配 inode 块(16 个/块)。两棵 B+ 树 + 128 项回收栈。

两层 B+ 树可管理 63×63 = 3969 个 inode，满足最大 8064 需求。

inode 分配算法
1尝试从回收栈弹出 inode 号（快速路径）
2  → 位置树查找物理位置 → 空闲树清除标记 → 清零 → 返回
3栈空 → 查父 inode 所在块组（慢速路径）
4  在空闲树中找该块组有空闲槽的 inode 块
5  找到 → 取空闲槽，清标记
6  未找到 → 分配新数据块作为 inode 块
7    初始化全零，空闲树注册(位图=0xFFFE)
8生成全局唯一 inode 编号
9位置树插入 (inode号, 块号, 槽位)
叶节点分裂（位置树满时）
1叶满(63项) → 从中间一分为二
2前半留原叶，后半写入新叶
3无索引层 → 创建新索引根，树升一层
4有索引层 → 父索引插入新叶指针

Extent 变长区段映射与自动合并

三级策略：

相邻合并算法
1输入：Extent 数组（按逻辑块号排序）
2标记"已变化"= 真
3当"已变化"为真时循环：
4  "已变化"= 假
5  对每对相邻记录 (i, i+1)：
6    若 i 的逻辑结束 = i+1 的逻辑起始
7    且 i 的物理结束 = i+1 的物理起始
8      → 合并：i 的长度 += i+1 的长度
9      → 删除 i+1，数组长度减一
10      → "已变化"= 真，重新扫描

效果：顺序写入的文件通常只保留 1 条 Extent 记录。内联查找 O(1)，B+ 树 O(log N)。
